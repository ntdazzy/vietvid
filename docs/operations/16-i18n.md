Cách Vyra phục vụ nhiều quốc gia trên cùng một codebase: chọn ngôn ngữ tự động theo IP, cho người dùng đổi tay, và ẩn/hiện nội dung theo vùng (vd câu "giọng Việt đỉnh cao" chỉ bật ở VN). VN trước, quốc tế sau.

Hiện app chỉ có **một** từ điển tiếng Việt (`apps/web/src/lib/i18n/vi.ts`), `<html lang="vi">` cứng, và phần lớn chữ vẫn **hardcode inline** trong component — chưa có `next-intl`, chưa có routing `/[locale]`, chưa có middleware.

## 1. Bức tranh tổng thể (đọc cái này trước)

Mục tiêu: một codebase, nhiều ngôn ngữ, **không** duplicate trang. Cơ chế "đỉnh nhất" hiện nay cho Next.js 14 App Router là **next-intl**.

| Mảnh | Vai trò | Trạng thái |
|---|---|---|
| `next-intl` | Thư viện i18n cho App Router: routing, dịch chuỗi server+client, format số/ngày/tiền | 🔜 chưa cài |
| Routing `/[locale]/...` | URL có tiền tố ngôn ngữ: `/vi/pricing`, `/en/pricing`. Tốt cho SEO + chia sẻ link | 🔜 |
| `middleware.ts` | Tự nhận locale theo **IP** (header geo) → fallback `Accept-Language` → cookie override → redirect | 🔜 |
| File dịch `messages/*.json` | Chuỗi dịch theo ngôn ngữ (vi/en/ja/zh/ko/es) | ⚙️ mới có `lib/i18n/vi.ts` (1 ngôn ngữ, không phải JSON) |
| Nút đổi ngôn ngữ | UI cho user tự chọn, ghi cookie `NEXT_LOCALE` | 🔜 |
| Nội dung theo vùng | Ẩn/hiện copy theo quốc gia (vd "giọng Việt" chỉ ở VN) | 🔜 |
| SEO `hreflang` | Báo Google bản dịch nào cho ngôn ngữ nào | 🔜 |

**Danh sách locale chốt:** `vi` (mặc định), `en`, `ja`, `zh`, `ko`, `es`.

**Nguyên tắc giữ trung thực:** Vyra mới ra mắt — **không** bịa "đã có ở 6 nước". i18n là **năng lực kỹ thuật sẵn sàng**, bật từng ngôn ngữ khi thực sự có người dùng/đối tác vùng đó. Bật `vi` + `en` trước; ja/zh/ko/es dịch dần (xem §8).

### 3.2 Khai báo config locale

Tạo `apps/web/src/i18n/routing.ts`:

```ts
import { defineRouting } from "next-intl/routing";

export const routing = defineRouting({
  // Thứ tự = ưu tiên hiển thị trong nút đổi ngôn ngữ.
  locales: ["vi", "en", "ja", "zh", "ko", "es"],
  defaultLocale: "vi",
  // "as-needed" = KHÔNG prefix cho defaultLocale (/, /pricing) nhưng prefix cho phần còn lại (/en, /ja/pricing).
  // Đổi sang "always" nếu muốn /vi/... luôn hiện (tốt cho SEO rõ ràng, xấu cho URL gốc VN).
  localePrefix: "as-needed",
});
```

> **Quyết định cần chốt:** `as-needed` (URL VN sạch `/pricing`, quốc tế có prefix) **hay** `always` (mọi ngôn ngữ đều `/vi/`, `/en/`). VN-first → `as-needed` hợp lý hơn cho thị trường chính. Xem §6 ảnh hưởng SEO.

Tạo `apps/web/src/i18n/navigation.ts` (wrapper Link/redirect biết locale):

```ts
import { createNavigation } from "next-intl/navigation";
import { routing } from "./routing";

export const { Link, redirect, usePathname, useRouter, getPathname } =
  createNavigation(routing);
```

> Sau này **mọi** `<Link>` nội bộ import từ `@/i18n/navigation` thay vì `next/link` — nó tự thêm prefix locale.

### 3.3 Loader chuỗi dịch

Tạo `apps/web/src/i18n/request.ts`:

```ts
import { getRequestConfig } from "next-intl/server";
import { routing } from "./routing";

export default getRequestConfig(async ({ requestLocale }) => {
  let locale = await requestLocale;
  if (!locale || !routing.locales.includes(locale as any)) {
    locale = routing.defaultLocale;
  }
  return {
    locale,
    messages: (await import(`../../messages/${locale}.json`)).default,

});
```

## 4. Routing `/[locale]` — di chuyển cây trang

App Router cần **bọc** toàn bộ trang trong segment `[locale]`. Đây là thay đổi lớn nhất.

### 4.1 Cấu trúc thư mục đích

```
apps/web/src/app/
  [locale]/
    layout.tsx          # layout gốc (lang={locale}, NextIntlClientProvider)
    page.tsx            # home
    pricing/page.tsx
    app/...             # toàn bộ trang app (dashboard/create/billing...)
    features/[key]/page.tsx
  globals.css           # giữ ngoài [locale]
```

> **Cảnh báo migrate:** đây là di chuyển hàng loạt file dưới `app/`. Làm khi `next dev` **đã tắt** (HANDOFF §6: build/move khi dev chạy → hỏng `.next`, trang trắng). Sau migrate: `tsc --noEmit` + live QA Playwright mọi màn (HANDOFF §8.2).

### 4.2 Layout gốc mới `app/[locale]/layout.tsx`

Chuyển nội dung từ `app/layout.tsx` hiện tại vào đây, đổi 2 chỗ:

```tsx
import { NextIntlClientProvider } from "next-intl";
import { getMessages, setRequestLocale } from "next-intl/server";
import { notFound } from "next/navigation";
import { routing } from "@/i18n/routing";
// ... giữ nguyên import font + Providers + globals.css

export function generateStaticParams() {
  return routing.locales.map((locale) => ({ locale }));
}

export default async function LocaleLayout({
  children,
  params,
}: {
  children: React.ReactNode;
  params: Promise<{ locale: string }>;
}) {
  const { locale } = await params;
  if (!routing.locales.includes(locale as any)) notFound();
  setRequestLocale(locale); // bật static render
  const messages = await getMessages();

  return (
    <html
      lang={locale}                                   // ← thay cho lang="vi" cứng
      className={`${display.variable} ${body.variable} ${numeric.variable} ${GeistMono.variable}`}
      suppressHydrationWarning
    >
      <body className="min-h-dvh antialiased">
        <NextIntlClientProvider messages={messages}>
          <Providers>{children}</Providers>
        </NextIntlClientProvider>
        <div className="grain" aria-hidden />
      </body>
    </html>
  );
}
```

> **Font tiếng Việt:** giữ `subsets: ["latin", "vietnamese"]` cho Be Vietnam Pro + Inter (đã có ở `app/layout.tsx`). Khi thêm `ja/zh/ko`, các font Latin **không** render được chữ Hán/Kana/Hangul → cần font phủ CJK (vd Noto Sans JP/SC/KR) nạp **có điều kiện theo locale** để khỏi tải nặng cho user VN. **Ước lượng, cần đo thật:** kích thước font CJK lớn (vài trăm KB–MB) → đo Lighthouse trước/sau khi bật.

### 4.3 Metadata đa-ngôn-ngữ

`app/layout.tsx` hiện hardcode `title`/`description` tiếng Việt. Chuyển sang `generateMetadata` đọc từ file dịch:

```tsx
import { getTranslations } from "next-intl/server";

export async function generateMetadata({
  params,
}: { params: Promise<{ locale: string }> }) {
  const { locale } = await params;
  const t = await getTranslations({ locale, namespace: "meta" });
  return { title: t("title"), description: t("description") };
}
```

## 5. Middleware: tự nhận ngôn ngữ theo IP + fallback + cookie

Đây là phần "đỉnh nhất" user yêu cầu: lần đầu vào, đoán đúng ngôn ngữ theo **quốc gia**; user đổi tay thì **nhớ** lựa chọn.

**Thứ tự ưu tiên (chốt):**
1. URL đã có prefix locale → tôn trọng (vd ai đó share `/ja/pricing`).
2. **Cookie `NEXT_LOCALE`** (user đã đổi tay) → ưu tiên cao nhất sau URL.
3. **Header geo theo IP** (`x-vercel-ip-country` trên Vercel / `cf-ipcountry` trên Cloudflare) → map quốc gia → ngôn ngữ.
4. **`Accept-Language`** của trình duyệt.
5. `defaultLocale` = `vi`.

Tạo `apps/web/src/middleware.ts`:

```ts
import createIntlMiddleware from "next-intl/middleware";
import type { NextRequest } from "next/server";
import { routing } from "@/i18n/routing";

const intlMiddleware = createIntlMiddleware(routing);

// Map quốc gia (ISO-3166 alpha-2) → locale Vyra hỗ trợ.
const COUNTRY_TO_LOCALE: Record<string, string> = {
  VN: "vi",
  JP: "ja",
  KR: "ko",
  CN: "zh", TW: "zh", HK: "zh",
  ES: "es", MX: "es", AR: "es", CO: "es",
  // còn lại → "en" (mặc định quốc tế)
};

export function middleware(req: NextRequest) {
  const hasCookie = req.cookies.has("NEXT_LOCALE");
  const url = req.nextUrl;
  const hasPrefix = routing.locales.some(
    (l) => url.pathname === `/${l}` || url.pathname.startsWith(`/${l}/`),
  );

  // Chỉ đoán theo IP khi: chưa có cookie VÀ URL chưa có prefix.
  if (!hasCookie && !hasPrefix) {
    const country =
      req.headers.get("x-vercel-ip-country") ||   // Vercel
      req.headers.get("cf-ipcountry") ||           // Cloudflare
      "";
    const guessed = COUNTRY_TO_LOCALE[country.toUpperCase()];
    if (guessed && guessed !== routing.defaultLocale) {
      // Gợi ý cho next-intl qua header — nó vẫn xét Accept-Language nếu không có.
      const res = intlMiddleware(req);
      return res;
      // (Phương án mạnh hơn: tự redirect tới `/${guessed}${path}` — xem ghi chú dưới.)
    }
  }

  // next-intl tự xử lý cookie + Accept-Language + defaultLocale.
  return intlMiddleware(req);
}

export const config = {
  // Bỏ qua API, file tĩnh, ảnh.
  matcher: ["/((?!api|_next|_vercel|.*\\..*).*)"],
};
```

> **Ghi chú quan trọng — IP vs Accept-Language:** next-intl **mặc định** chỉ dùng `Accept-Language` + cookie, **không** tự đọc header geo. Để IP thắng `Accept-Language` (vd người Việt cài máy tiếng Anh nhưng đang ở VN → muốn ra `vi`), cần **tự redirect**: khi `guessed` khác default và URL chưa prefix, trả `NextResponse.redirect(new URL('/' + guessed + path, req.url))`. Cân nhắc: redirect dựa IP có thể gây vòng lặp nếu logic sai → **luôn** kèm điều kiện `!hasPrefix && !hasCookie`. Test kỹ bằng giả lập header (§5.1).

> **Header geo chỉ có ở prod.** Local dev **không** có `x-vercel-ip-country`/`cf-ipcountry` → middleware rơi về Accept-Language. Đừng kết luận "geo hỏng" khi test máy local.

## 6. Nút đổi ngôn ngữ (UI)

Đặt vào header marketing (`components/marketing/site-header.tsx`) + top-bar app (`components/shell/top-bar.tsx`). Khi đổi, ghi cookie `NEXT_LOCALE` và điều hướng giữ nguyên path.

```tsx
"use client";
import { useLocale } from "next-intl";
import { usePathname, useRouter } from "@/i18n/navigation";
import { routing } from "@/i18n/routing";

const LABEL: Record<string, string> = {
  vi: "Tiếng Việt", en: "English", ja: "日本語",
  zh: "中文", ko: "한국어", es: "Español",
};

export function LocaleSwitcher() {
  const locale = useLocale();
  const pathname = usePathname();
  const router = useRouter();

  function change(next: string) {
    // Ghi cookie để middleware tôn trọng ở các lần sau (1 năm).
    document.cookie = `NEXT_LOCALE=${next};path=/;max-age=31536000;samesite=lax`;
    router.replace(pathname, { locale: next });
  }

  return (
    <label className="sr-only-wrapper">
      <span className="sr-only">Ngôn ngữ</span>
      <select
        aria-label="Đổi ngôn ngữ"
        value={locale}
        onChange={(e) => change(e.target.value)}
        className="glass-bordered rounded-md bg-transparent px-2 py-1 text-sm text-ink-medium"
      >
        {routing.locales.map((l) => (
          <option key={l} value={l}>{LABEL[l]}</option>
        ))}
      </select>
    </label>
  );
}
```

> Tuân design system (HANDOFF §9): dùng `.glass-bordered`, màu `ink-medium`, có `aria-label`. KHÔNG bịa cờ quốc gia loè loẹt — chữ tên ngôn ngữ là đủ và chuẩn a11y.

## 8. Nội dung theo vùng (region-aware copy)

Yêu cầu cụ thể của chủ dự án: câu **"giọng Việt đỉnh cao"** chỉ có ý nghĩa với khách VN. Không hiển thị nó cho user Nhật/Mỹ.

Cách làm: **đừng** dựa duy nhất vào ngôn ngữ (user VN có thể đang xem bản `en`). Tách 2 khái niệm:
- **Ngôn ngữ (`locale`)** = chữ hiển thị.
- **Vùng (`country`)** = từ header geo, quyết định *nội dung nào phù hợp*.

Truyền `country` xuống component (đọc trong Server Component qua `headers()`):

```tsx
import { headers } from "next/headers";

export default async function HomePage() {
  const h = await headers();
  const country = (h.get("x-vercel-ip-country") || h.get("cf-ipcountry") || "").toUpperCase();
  const isVN = country === "VN";
  return (
    <>
      <Hero />
      {/* Wedge "giọng Việt thật" chỉ bật ở VN */}
      {isVN && <VietnameseVoiceWedge />}
    </>
  );
}
```

> **Cẩn thận cache:** Server Component đọc `headers()` → trang thành **dynamic** (không static). Với trang marketing cần SEO tĩnh, cân nhắc: hoặc chấp nhận dynamic, hoặc tách block region-aware thành Client Component đọc cookie/locale. **Ước lượng, cần đo thật:** ảnh hưởng TTFB của dynamic rendering — đo bằng Lighthouse + Vercel Analytics sau khi bật.

> Quy tắc trung thực (HANDOFF §8.1): copy theo vùng **không được bịa**. "Giọng Việt thật" đúng với năng lực thật ở VN; đừng dịch nguyên si thành "best Vietnamese voice" cho user Mỹ vì với họ nó vô nghĩa — thay bằng wedge khác (vd "transparent credits", "100% refund on system error").

## 9. SEO: hreflang + canonical

Báo Google: mỗi trang có những bản dịch nào, ngôn ngữ nào. Tránh bị xem là nội dung trùng lặp.

next-intl + `generateMetadata` có thể sinh `alternates.languages`:

```tsx
import { getTranslations } from "next-intl/server";
import { routing } from "@/i18n/routing";

export async function generateMetadata({ params }: { params: Promise<{ locale: string }> }) {
  const { locale } = await params;
  const t = await getTranslations({ locale, namespace: "meta" });
  const base = "https://vyra.vn"; // domain prod thật — chốt sau

  const languages: Record<string, string> = {};
  for (const l of routing.locales) {
    languages[l] = l === routing.defaultLocale ? `${base}/pricing` : `${base}/${l}/pricing`;
  }
  languages["x-default"] = `${base}/pricing`; // bản mặc định cho ngôn ngữ chưa map

  return {
    title: t("title"),
    description: t("description"),
    alternates: { canonical: languages[locale], languages },
  };
}
```

Kết quả HTML mong muốn:

```html
<link rel="alternate" hreflang="vi" href="https://vyra.vn/pricing" />
<link rel="alternate" hreflang="en" href="https://vyra.vn/en/pricing" />
<link rel="alternate" hreflang="x-default" href="https://vyra.vn/pricing" />
```

> **Domain prod chưa chốt** — `vyra.vn` ở trên là placeholder. Lấy domain thật khi cấu hình DNS (xem [02-infrastructure.md](02-infrastructure.md), [17-deployment-cicd.md](17-deployment-cicd.md)) rồi đưa vào biến env `NEXT_PUBLIC_SITE_URL` thay vì hardcode.

**Kiểm chứng SEO sau deploy:**
```bash
curl -s https://vyra.vn/en/pricing | grep -i hreflang   # phải thấy đủ 6 ngôn ngữ + x-default
```
Và dùng Google Search Console → International Targeting để xác nhận không có lỗi hreflang.

## 10. Tương tác với backend (app_api)

i18n hiện là **frontend-only**. Backend `app_api/` không cần đa-ngôn-ngữ ngay vì:
- Lỗi API trả mã/khoá (vd `INSUFFICIENT_CREDITS`), frontend map sang chữ dịch.
- Nội dung video (giọng Việt) là sản phẩm, không phải UI.

🔜 **Khi cần sau:** nếu gửi email giao dịch/biên nhận đa-ngôn-ngữ, lưu `preferred_locale` của user vào DB (cột mới trên `users`/membership) và truyền vào template email. Chưa làm — chỉ ghi để session sau biết.

> **Giá theo vùng/tiền tệ** (USD cho quốc tế vs VND cho VN) là chuyện của [08-pricing-plans-credits.md](08-pricing-plans-credits.md) + [10-payments.md](10-payments.md), không phải file này. i18n chỉ lo *hiển thị* số đã format (next-intl `useFormatter().number(x, {style:'currency', currency})`).

## 11. Thứ tự triển khai khuyến nghị (đừng làm hết một lần)

1. **Cài next-intl + routing vi/en** (chỉ 2 ngôn ngữ) → migrate `vi.ts` → `messages/vi.json` → dịch `en.json`.
2. **Bọc `[locale]`** + sửa layout (`lang={locale}`, provider, metadata). Verify mọi màn.
3. **Middleware geo** (VN→vi, còn lại→en) + nút đổi ngôn ngữ + cookie.
4. **hreflang** + canonical với domain thật.
5. **Region-aware copy** (ẩn wedge "giọng Việt" ngoài VN).
6. **Thêm ja/zh/ko/es** khi có nhu cầu thật (font CJK + bản dịch người-duyệt).
