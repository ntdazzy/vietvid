## P0 — đã FIX + verify (2 lỗi gốc khiến app "không map")

### P0-1: Header luôn hiện "Sign in" dù đã đăng nhập (trên trang marketing)
- **Triệu chứng:** user đăng nhập rồi vào `/` → header vẫn nút "Sign in" → tưởng bị đăng xuất / chưa map.
- **Gốc rễ:** `apps/web/src/components/marketing/site-header.tsx:67` — `me` query + toàn bộ UI authed dựa vào **prop `authed` cứng**. App-layout truyền `authed=true`, nhưng trang marketing (`/`, `/pricing`, feature...) render `SiteHeader` mặc định `authed=false` → luôn render nút đăng nhập (dòng 167), bất kể có phiên thật.
- **Sửa:** header nhận trạng thái đăng nhập THẬT — thêm `signedIn` đọc `localStorage[vietvid_dev_token]` / cookie `vyra_auth`, `isAuthed = authed || signedIn`, dùng `isAuthed` cho mọi nhánh + bật `me` query theo `isAuthed`.
- **Kiểm chứng:** đăng nhập → `/` → header `bangDieuKhien=true`, `dangNhap=false`, body "...Vyra **Bảng điều khiển** KOL AI Tạo nội dung..."; chưa đăng nhập → `/` → `dangNhap=true`. ✓

### P0-2: UI hiện tiếng Anh ("Sign in") trên localhost
- **Triệu chứng:** chữ "Sign in / Home / Tools" tiếng Anh — user VN tưởng lỗi.
- **Gốc rễ:** `apps/web/src/middleware.ts:44` — localhost không có header geo (`x-vercel-ip-country`) → đặt `NEXT_LOCALE=en`.
- **Sửa:** không có geo (localhost/proxy) HOẶC VN → `vi`; chỉ nước khác → `en`.
- **Kiểm chứng:** context mới → `/` và `/app` render tiếng Việt ("Bảng điều khiển", "Tạo nội dung", "Tạo video", "VI"). ✓
- **Lưu ý user:** trình duyệt đang có cookie `NEXT_LOCALE=en` cũ → bấm nút đổi ngôn ngữ (VI) 1 lần hoặc xoá cookie; khách mới mặc định vi.
